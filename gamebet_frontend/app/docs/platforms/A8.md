# A8 聚合采集框架

A8 桌面端通过 Socket.IO 向浏览器推送多平台赔率；本仓库用同一套 accumulator 落库。

## 目录结构

| 文件 | 作用 |
|------|------|
| `a8/socketHub.ts` | 连接 `DEFAULT_A8_WS`，`join room` 订阅频道，多采集器共享单连接 |
| `a8/betsCollect.ts` | `A8BetsCollector`：ingest 消息 → `oddsStore` + 内存赛事/盘口 |
| `a8/index.ts` | `startA8BetsCollector`：订阅频道 + 60s 批量 `saveMatch` / `saveBets` |
| `a8/pluginBridge.ts` | Stake 等通过 A8 插件代理 HTTP（`a8PluginPost`） |

## 消息形态 `A8BetsMessage`

```ts
{
  matchId?: string | number;
  homeName?: string;
  awayName?: string;
  gameId?: string | number;
  bets?: A8BetRow[];  // betId, home/away 赔率, map, name, locked, sportId...
}
```

## 赔率 ID 规则

| 模式 | `homeSuffix` / `awaySuffix` | 说明 |
|------|----------------------------|------|
| 默认 | `"1"` / `"2"` | `homeId = ${betId}:1`，`awayId = ${betId}:2`（IM、XBet 主客） |
| `useDirectIds: true` | — | 使用推送里的 `homeId` / `awayId`（Stake） |

## 落库节奏

- 每 **5s** 轮询检查是否该保存
- 距上次成功保存 **≥ 60s** 时组包；仅当用户中心该平台**回传开关**打开时，`saveMatch` / `saveBets` 才会请求后端（`collectStore` 内 gate）

## 使用此框架的平台

- [IM.md](./IM.md) — 频道 `IM`
- [XBet.md](./XBet.md) — 频道 `XBet` + `XBet:Score`
- [Stake.md](./Stake.md) — 频道 `Stake` + GraphQL 快照 `mergeGraphqlMatch`
