copy 项目 — 模块化 CSS 取代 a8.css
================================================================

目标
  public/copy/styles/ 是 changmen 自有样式工程。
  验收：本地 DEV 在 `/` 切换 modules / legacy 皮肤（同一 Vue + 真实数据）

边界（硬约束）
  在 modules 完全取代 legacy/a8 并通过走查之前：
    · 生产 build 始终 src/styles/* bundle
    · DEV 才可用 modules 栈（右下角角标或 ?a8=0|1）
  走查通过后：bootstrap 改引 modules，删 DEV 切换

目录
  styles/index.css          分桶 @import 清单（extract 维护）
  styles/modules/         主皮肤分桶 + segments/（extract 产出，gitignore）
  styles/modules/changmen/  补丁层（sync 自 src/styles，进 git）
  styles/legacy/*.css     从 src/styles 同步（gitignore，HEAD 自检用）

首次 / legacy 更新后：
  cd changmen\apps\web
  node public/copy/sync-styles.mjs
  node scripts/extract-a8-to-modules.mjs

使用
  npm run app:dev  → http://localhost:5174/
  默认 DEV modules 皮肤；角标切 legacy = 与生产 build 相同 bundle
  ?a8=1 legacy · ?a8=0 modules
  /copy 已重定向到 /

自检：
  npm run verify:copy-modules
  DEV 右下角角标 · styles ok

迁移顺序（建议）
  1. tokens + layout（分桶编辑）
  2. platform-icons
  3. sidebar-user / sidebar-orders
  4. account-bar / match-list / bet-row
  5. extension-banner / dialogs
  6. element-* 分桶
  7. changmen 补丁齐全；modules 零 legacy
  8. a8-remainder 已清空（0 条）
  9. 走查通过后：生产 bootstrap 改引 modules，删 DEV 角标
